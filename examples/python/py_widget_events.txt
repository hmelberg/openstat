#%% md
# Events on HTML elements

`ui.on(selector, event, function, target=…)` binds a python function to
an HTML event; the return value is rendered in the `target` element
(text → `<pre>`, DataFrame → table, plotly figure → chart). If `target`
is omitted, the result is appended to the cell's own output slot.
`ui.run_cell(selector, event, cell_id)` instead reruns a named cell —
handy when you want to reuse a whole plot cell from a button.

This is the **push model**: the button pushes an event into Python,
unlike `ui.slider`/`ui.dropdown`, which are pull (the cell reads the
value).

#%% html
<button id="calcBtn">Compute mean</button>
<button id="plotBtn">Redraw plot</button>
<div id="result"></div>

#%% python
import ui

def calculate(evt):
    return iris.groupby("species")["sepal_length"].mean().to_frame()

ui.on("#calcBtn", "click", calculate, target="result");
ui.run_cell("#plotBtn", "click", "plot");

#%% python id=plot
import plotly.express as px
px.scatter(iris, x="sepal_length", y="petal_length", color="species")
