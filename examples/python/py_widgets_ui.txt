#%% md
# Widgets med `ui`

`ui`-modulen gir kontroller (glidebryter, nedtrekksmeny, knapp m.fl.) som
lever i cellens output-slot, sammen med resultatet under. Dette er en
**pull-modell**: verdien kontrollen har akkurat nå ligger lagret i
nettleseren (ikke i Python), og hvert `ui.*`-kall i cellen leser den ut
igjen. Endrer du en kontroll, kjøres cellen (eller en annen navngitt celle
via `rerun=`) på nytt automatisk — scriptet leser da ut den nye verdien
neste gang det kaller `ui.*`. Legg til `placement="left"` (eller
`"top"`/`"bottom"`) på et enkelt `ui.*`-kall for å plassere akkurat den
kontrollen i en annen stripe enn cellens standard.

#%% python
import ui

n = ui.slider(1, 20, value=5, label="Antall rader")
art = ui.dropdown(sorted(iris["species"].unique()), label="Art")

iris[iris["species"] == art].head(n)

#%% python id=plot
# Denne cellen BRUKER widget-verdiene (n og art fra cellen over — cellene
# deler sesjon), men har ingen egne widgets: den oppdateres derfor ikke av
# seg selv når du endrer kontrollene. Det er Oppdater-knappen under som
# kjører den på nytt (rerun="plot" peker på id-en i denne cellens header).
import plotly.express as px

sub = iris[iris["species"] == art].head(n)
px.scatter(sub, x="sepal_length", y="petal_length",
           title=f"Iris — {art}, første {n} rader")

#%% python
import ui

ui.button("Oppdater", rerun="plot")
