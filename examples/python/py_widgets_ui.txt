# Widgets with `ui`

The `ui` module provides controls (slider, dropdown, button, and more)
that live in the cell's output slot, alongside the result below. This
is a **pull model**: the control's current value is stored in the
browser (not in Python), and each `ui.*` call in the cell reads it back
out. When you change a control, the cell (or another named cell via
`rerun=`) reruns automatically — the script then reads out the new
value the next time it calls `ui.*`. Add `placement="left"` (or
`"top"`/`"bottom"`) to a single `ui.*` call to place that particular
control in a different strip than the cell's default. If you need one
shared run button in the strip, `ui.run_button(target=...)` gives you
that — `target="all"` (default) runs the whole document, a cell id
runs only that cell.

#%% python
import ui

n = ui.slider(1, 20, value=5, label="Number of rows")
species = ui.dropdown(sorted(iris["species"].unique()), label="Species")

iris[iris["species"] == species].head(n)

#%% python id=plot
# This cell USES the widget values (n and species from the cell above —
# cells share a session), but has no widgets of its own: it therefore
# doesn't update by itself when you change the controls. It's the
# ui.run_button below that reruns it (target="plot" points at the id in
# this header).
import plotly.express as px

sub = iris[iris["species"] == species].head(n)
px.scatter(sub, x="sepal_length", y="petal_length",
           title=f"Iris — {species}, first {n} rows")

#%% python
import ui

ui.run_button("Update", target="plot")
