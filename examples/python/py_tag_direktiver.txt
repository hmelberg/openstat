# label: Notebook — #tag directives and content sniffing
#options.mode = python
#options.title = "Cell directives with #tag"
#options.description = "#tag.key = value in the cell body: metadata that survives Colab/Jupytext round-trips, defaults in the preamble, and cells that reveal their own type"
#tag.type = python

#%%
"""
# `#tag` directives

Metadata can live as comment lines at the TOP of the cell body instead
of on the `#%%` line: `#tag.key = value`, one per line. Colab and
Jupytext own the `#%%` lines and rewrite them — comment lines in the
body always survive. This cell is also *sniffed*: it's a single
triple-quoted string on its own (three double quotes), and is therefore
rendered as markdown without any `md` marker.

- `#tag.type = python` in the preamble above sets the document's default cell type.
- `#tag.hide-code = true` below hides the cell's code, just like the header flag.
- Everything is still shown as plain text in the editor — raw text IS the editor.
"""

#%%
#tag.hide-code = true
#tag.id = calculation
total = sum(range(10))
total

#%%

#tag.id = table
select 42 as answer
