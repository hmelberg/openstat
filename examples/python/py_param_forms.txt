#%% md
# #@param forms

Notebooks from Google Colab that use the `#@param` comment work
**directly** here — no conversion needed. A line like

```python
n = 5  #@param {type:"slider", min:0, max:20}
```

gets a control drawn above the cell; change the control, and the new
value is written straight into **the source text itself** (check
«Raw text» — it's always the text, not hidden state, that is the
source of truth).

**`run:"auto"` vs. manual:** add `run:"auto"` to the metadata object,
and the cell reruns automatically every time the control changes.
Without `run:"auto"`, only the text is written, and the cell gets the
usual "changed, not run" marker (stale tint) until you press ▶
yourself.

`#@title` and `#@markdown` (Colab's cell-title/text comments) are
**not supported yet** — coming in a later round.

Add `placement:"left"` (or `"top"`/`"bottom"`) to the metadata object
to place that particular control in a different strip than the cell's
default.

**R is also supported:** `x <- 3  #@param {type:"slider", min:0, max:10}`
— the parser recognizes both `=` and R's `<-` as assignment operators.

#%% python
n = 5  #@param {type:"slider", min:0, max:20, step:1, run:"auto"}
iris.head(n)

#%% python
species = 'setosa'  #@param ["setosa", "versicolor", "virginica"]
iris[iris["species"] == species]

#%% python
all_columns = True  #@param {type:"boolean"}
iris if all_columns else iris[["sepal_length", "petal_length", "species"]]

#%% python
color = 'blue'  #@param ["blue", "red", "green"] {allow-input:true}
f"Favorite color: {color}"

#%% python
date = '2026-07-15'  #@param {type:"date"}
f"Selected date: {date}"
