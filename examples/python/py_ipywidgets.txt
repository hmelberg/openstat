# ipywidgets — real Jupyter widgets

The `ui` module (see the example «Widgets — ui.slider/dropdown») is a
simple **pull** model that lives in the browser. This notebook instead
shows the **bridge** to real `ipywidgets` — the same library (and the
same comm protocol) as in genuine Jupyter or Google Colab. The controls
now live as real Python objects in the kernel (traitlets), and values
sync in BOTH directions between browser and Python — not just read back
on the next run.

The first time a document runs `import ipywidgets`, pyodide installs a
pinned package set (ipywidgets + IPython and others, **~10 MB**) and
loads a pinned frontend bundle — this happens only once per session (and
only for documents that actually import `ipywidgets`; a plain script
without that import loads nothing extra).

**v1 scope:** standard controls (slider, dropdown, ...), `observe` and
two-way value sync, display in the cell's output. **Not supported yet:**
`interact()` and the `Output` widget (planned later) — so `observe`
callbacks below run OUTSIDE a captured cell: `print(...)` inside a
callback ends up in the browser console, not in the cell's output.

#%% python
import ipywidgets as w

n = w.IntSlider(value=5, min=0, max=20, description="Count")
n

#%% python
share = w.FloatSlider(value=2.5, min=0, max=10, step=0.1, description="Share")
share

#%% python
species = w.Dropdown(
    options=["setosa", "versicolor", "virginica"],
    description="Species",
)
species

#%% md
### `observe` — Python reacts to changes in real time

`observe` registers a callback that runs in Python EVERY time the
control above (`n`) changes value in the browser — no cell is rerun.
The callback below updates a `Label`'s text; since the `Label` is
already displayed (the cell has run), the visible text updates LIVE as
you drag the sliders above.

#%% python
status = w.Label(value=f"Count is now: {n.value}")

def _show_count(change):
    status.value = f"Count is now: {change['new']}"
    print("observe callback ran:", change["new"])  # ends up in the console (v1)

n.observe(_show_count, names="value");
status

#%% md
### Python reads the control's value (kernel sync)

The control's value lives in the Python object `n` — reading `n.value`
gives exactly the value the slider shows right now (drag it above,
rerun the cell below, and watch it follow along).

#%% python
n.value

#%% md
### One step: create, change, display

A common idiom: create a control, change it right away, and display it
— all in the same cell. The bridge buffers any changes that arrive
before the frontend registration finishes, so the order is always
correct: the slider below should show **2**, not the initial value 5.

#%% python
s = w.IntSlider(5)
s.value = 2
s
