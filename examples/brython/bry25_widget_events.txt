# Events on HTML elements (Brython)

`ui.on(selector, event, function, target=…)` binds a brython function to
an HTML event; the return value is rendered in the `target` element (text →
`<pre>`, DataFrame → table, plotly figure → chart). If `target` is omitted,
the result is appended to the cell's own output slot. `ui.run_cell(selector,
event, cell_id)` reruns a named cell instead.

This is the **push model**: the button pushes an event into Brython, unlike
`ui.slider`/`ui.dropdown`, which are pull (the cell reads the value).

#%% html
<button id="beregnBtn">Show rows</button>
<button id="plotBtn">Redraw plot</button>
<div id="resultat"></div>

#%% brython
import ui

def show_rows(evt):
    return iris.head(5)

ui.on("#beregnBtn", "click", show_rows, target="resultat")
ui.run_cell("#plotBtn", "click", "plot")

#%% brython id=plot
import plotly_express_brython as pe
pe.scatter(iris, x="sepal_length", y="petal_length", color="species")
