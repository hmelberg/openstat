#%% md
# Widgets med `ui_*()` i R

R kan ikke spørre nettleseren om en verdi midt i kjøringen (webR kjører i en
egen worker), så R-fasaden bruker en **deklarer-og-injiser**-modell i stedet
for pythons pull-modell: `ui_slider()` m.fl. *deklarerer* en kontrollspec, og
returnerer verdien som allerede ligger lagret fra forrige kjøring (eller sin
egen default første gang). Kontrollene tegnes idet cellen kjører — enten via
per-celle ▶ eller «Kjør alle» — og det er selve kjøringen som leser
deklarasjonene og bygger stripa. Endrer du en kontroll, kjøres cellen (eller
en annen navngitt celle via `rerun=`) automatisk på nytt, med den nye verdien
injisert før kjøring. Legg til `placement = "left"` (eller `"top"`/`"bottom"`)
i et `ui_*()`-kall for å plassere akkurat den kontrollen i en annen stripe enn
cellens standard.

#%% r
n <- ui_slider(1, 10, value = 3, label = "Multiplikator")
n * 100

#%% r
# R forstår bare engelske fargenavn i col= — verdiene må derfor være
# gyldige R-farger (norske etiketter kan mappes med switch() om ønskelig).
farge <- ui_dropdown(c("steelblue", "firebrick", "seagreen"), label = "Farge")

#%% r id=plot
plot(iris$sepal_length, iris$petal_length,
     col = farge, pch = 19,
     main = paste("Iris — kronblad mot begerblad (n =", n, ")"),
     xlab = "sepal_length", ylab = "petal_length")

#%% r
ui_run_button("Oppdater", target = "plot")
