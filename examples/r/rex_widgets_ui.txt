# Widgets with `ui_*()` in R

R cannot ask the browser for a value mid-run (webR runs in its own worker),
so the R facade uses a **declare-and-inject** model instead of Python's pull
model: `ui_slider()` and friends *declare* a control spec, and return the
value already stored from the previous run (or their own default the first
time). The controls are drawn as the cell runs — either via per-cell ▶ or
"Run all" — and it is the run itself that reads the declarations and builds
the strip. If you change a control, the cell (or another named cell via
`rerun=`) is automatically rerun, with the new value injected before running.
Add `placement = "left"` (or `"top"`/`"bottom"`) to a `ui_*()` call to place
that particular control in a different strip than the cell's default.

#%% r
n <- ui_slider(1, 10, value = 3, label = "Multiplier")
n * 100

#%% r
# R only understands English color names in col= — the values must
# therefore be valid R colors (Norwegian labels could be mapped with switch() if desired).
color <- ui_dropdown(c("steelblue", "firebrick", "seagreen"), label = "Color")

#%% r id=plot
plot(iris$sepal_length, iris$petal_length,
     col = color, pch = 19,
     main = paste("Iris — petal vs sepal (n =", n, ")"),
     xlab = "sepal_length", ylab = "petal_length")

#%% r
ui_run_button("Update", target = "plot")
