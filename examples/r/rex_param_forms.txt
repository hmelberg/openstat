#%% md
# `#@param` forms in R

A `#@param` comment after an assignment turns the line into a form field:
the form **writes the new value straight into the source line** (check via
"Raw text" if you like) and reruns the cell when the meta has `run:"auto"`
— without `run:"auto"` the cell is marked as changed (stale), and you run it
yourself with ▶. Both assignment operators work (`<-` and `=`), and R
literals are written correctly (`TRUE`/`FALSE`, numbers without quotes).
Notebooks made in Google Colab with `#@param` therefore also work here.

#%% r
n <- 50 #@param {type:"slider", min:10, max:150, step:10, run:"auto"}
sample_rows <- iris[seq_len(n), ]
paste("Sample:", nrow(sample_rows), "rows — mean sepal_length:",
      round(mean(sample_rows$sepal_length), 2))

#%% r
# Without run:"auto": the change is written into the text and the cell goes stale — run it yourself.
color <- 'steelblue' #@param ["steelblue", "firebrick", "seagreen"]
show_grid <- TRUE #@param {type:"boolean"}
{
  plot(sample_rows$sepal_length, sample_rows$petal_length,
       col = color, pch = 19,
       main = paste("Iris — first", n, "rows"),
       xlab = "sepal_length", ylab = "petal_length")
  if (show_grid) grid()
}

#%% md
Tip: `placement:"left"` in the meta places exactly that field in a column to
the left of the result — e.g.
`threshold <- 5 #@param {type:"number", placement:"left"}`.

Note: "Run all" evaluates R statement by statement — keep multi-step plots
(`plot()` + `grid()`/`abline()` etc.) together in one `{ }` block, like the
cell above does, otherwise later plot statements fail with "plot.new has
not been called yet".
