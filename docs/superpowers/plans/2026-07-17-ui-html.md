# ui.html + #tag.import + Widget Callables Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `ui.html.*` element builders (JS-side element engine, python id-handles), last-expression/`.show()` mounting, `on_change=callable` on widgets, `ui.value(name)`, and repeatable `#tag.import` with `ui.sl`/`ui.pico` shortcuts — per `docs/superpowers/specs/2026-07-17-ui-html-design.md`.

**Architecture:** js/ui.js gains an element engine (`Ui.el*`: id-registry, create/append/props/on/show; props applied property-else-setAttribute; mounts via the running run-ctx slot or the W5 target semantics) plus `Ui.value(name)` and a widget-callable channel (`data-ui-key` identity; a bound handler replaces the rerun on change). Facades add the `ui.html` namespace (pure-python kwarg normalization; wrapper holds the element id), callable dispatch in `on_change`/`on_click`, and `ui.value`. The engines' last-expression display paths get one duck-typed branch (`_openstat_el_id`) that mounts instead of printing. index.html consumes preamble `#tag.import` entries (via the exported `Cells.scanTagBlock`) and loads module scripts/stylesheets idempotently before user code.

**Tech Stack:** ES5 var-style JS, python facade twins, node test runner, pytest (FakeUiJs pattern).

## Global Constraints

- Pull model + all existing widget behavior unchanged; `on_change="cellnavn"` string form untouched (callable is additive; a callable control does NOT also rerun).
- Facade twins byte-mirror except documented dialect differences (pyodide `create_proxy`; mpy no stdout-swap — reuse each file's existing `_make_event_wrapper` plumbing for callables).
- Trust model unchanged (run-gate; static htmlTrusted untouched). mdRunNotebookCell contract untouched. Hybrid machinery untouched.
- ES5/Norwegian/`t()`+en.js. ANY index.html inline-script change gets a browser smoke IN THE SAME TASK; cross-IIFE only via `window.md*`.
- Baselines at plan start: node full 564/568 + 4 known ENOENT (cells-dom 99, cells 124, ui-dom 71, param-forms-dom 44, ui.test 50); pytest full 1247 + 1 known error (`tests/test_ui_module.py` 49, twin suites 47/48).
- R mode: polite notices only (Task 5); never a crash.

### Task 1: js/ui.js — element engine, `Ui.value`, widget-callable channel

**Files:** Modify `js/ui.js`; Test `tests/js/ui-dom.test.js` + `tests/js/ui.test.js` (append)

**Interfaces (produced — Tasks 2-4 depend on these exact names):**
- `Ui.elCreate(tag, propsJson)` → `elId` (string). propsJson: `{"props": {name: value…}, "style": {…}|string, "attrs": {…}, "events": ["click", …]}` — names arrive PRE-normalized from python (class, data-x, aria-label…); JS applies: DOM property when `name in node`, else `setAttribute` (dict/list values JSON-encoded; boolean true → empty-attr present, false → removed); `style` string → cssText, object → per-prop; failures `console.warn`, never throw.
- `Ui.elSetProps(elId, propsJson)` → same application on an existing node.
- `Ui.elAppend(parentId, childJson)` — childJson `{"el": elId}` or `{"text": "…"}`; `Ui.elClear(elId)`.
- `Ui.elOn(elId, event, handler)` — element-scoped binding reusing the W5 dispatch: register with key `el::<elId>::<event>` in `_bindings` (kind 'fn'), matched by elId (the element gets `data-ui-el="<elId>"`; the delegated listener's closest-match must also try `[data-ui-el]` bindings — extend `_installDelegate`'s matching: selector-bindings via `closest(b.selector)` as today, el-bindings via `closest('[data-ui-el="'+b.elId+'"]')`). Handler results render via the existing `renderEventResult` (binding carries cellIdx/target null → slot fallback).
- `Ui.elShow(elId, optsJson)` — optsJson `{"target": "dom-id"|null}`. target null: append the node to the RUNNING context's slot (`mdUiRunCtx()` → cellEl's `.nb-output-body`; doc ctx → `#outputArea`'s doc strip-area equivalent — append to `#outputArea` end; no ctx → console.warn + no-op). target set: replace-into-`#target` with a per-`(cellKey,target)` registry entry so re-running the declaring cell replaces (mirror the binding-lifecycle semantics; sweep entries in `endCellRun` for the cell, and in `resetDocument`).
- Element registry `_els = {}` (elId → node), ids `el<n>` monotonic; cleared in `Ui.resetDocument`. `Ui.elNode(elId)` → the raw node (the facades' `.el` escape hatch).
- `Ui.value(name)` → the stored value of the control whose key ends `::<name>` (last-registered wins on duplicates + one console.warn); undefined → null (JSON'able).
- Widget callables: `Ui.bindControlHandler(controlKeyJson…)` — simplest contract: `registerControl` returns, ALONGSIDE the value, the controlKey when the spec has `"has_handler": true` (extend the return to a JSON object `{value, key}` ONLY for has_handler specs — legacy return unchanged otherwise); then the facade calls `Ui.bindControlHandler(key, wrappedHandler)`. `_wireChange`: when `_controlHandlers[key]` exists, fire `handler(JSON.stringify({value: v}))` (through the same drop-while-running + error-wrap as `_dispatchBinding`) INSTEAD of `fireDebounced()`; `_syncPush` still runs first. Buttons: click fires handler instead of `_rerunFor`. Handlers destroyed on sweep/reset (guarded `.destroy`). Built control inputs also get `data-ui-key` (identity, spec requirement).
- Handler-result rendering for control handlers reuses `renderEventResult` with a synthetic binding `{cellIdx, target: null}`.

Steps: failing node tests (element create/props matrix incl. bool/dict-JSON/style both forms; append/clear; elShow slot-append via stubbed mdUiRunCtx + target-replace + rerun-replace; elOn fires and renders; Ui.value incl. duplicate warn; control handler fires with value, NO rerun, sync_to still pushed; sweep destroys handlers) → implement → full suite → commit `feat(ui): element-motor (Ui.el*), Ui.value og widget-callable-kanalen`.
