Privacy Statement

Microdata Script Runner — information about data processing for logged-in users.

Last updated: 2026-04-27 · Version: draft for review by the data protection officer.

Draft. This statement is a first draft and has not yet completed legal review. The final version will be published after review by the data protection officer (personvernombud).

1. What is this?

Microdata Script Runner is a hobby project run by Hans Olav Melberg that helps users write and run code against microdata.no syntax, Python and R directly in the browser. The editor itself requires no login and stores no data about you. If you choose to log in to use the AI assistant or save scripts online, personal data is processed as described below.

2. What information is collected?

For logged-in users, the following is stored:

What is not sent to the server: content you write in the editor while working locally, datasets you load, run results, output, or widget content. These remain in your browser.

3. Purpose and legal basis

The processing is necessary to deliver the service you have requested (Article 6(1)(b) of the GDPR — performance of a contract). You have given consent by logging in and accepting this statement.

4. Where is the data stored?

  • Account information, scripts and logs: Anvil Works (EU region). Anvil is the data processor.
  • AI queries during processing: Sent to Anthropic (USA) so the AI model can respond. Anthropic is the data processor. The query is not permanently stored by Anthropic; the response is stored by us for 90 days.
  • Login: The Microsoft Identity Platform handles SSO. Microsoft receives no service-specific information beyond the fact that you have logged in.

5. How long is the data stored?

  • Questions to the AI and generated script: 90 days, after which the content is automatically deleted. Aggregated counters (number of calls, model, latency) are retained without content data.
  • Saved scripts: Until you delete them or your account.
  • Account information: Until the account is deleted.
  • Audit logs: 12 months.
  • Session tokens: 30 days from last use.

6. Your rights

You have the right to:

  • Access — request a copy of all data stored about you
  • Rectification — have inaccurate information corrected
  • Erasure — request that all data about you be deleted (see below)
  • Data portability — have your saved scripts exported in a readable format
  • Objection and restriction — object to or restrict the processing
  • Complaint — you may lodge a complaint with the Norwegian Data Protection Authority (Datatilsynet) (datatilsynet.no)

7. Account deletion

You can request deletion by sending an email to the contact address at the bottom. Deletion is carried out within 30 days and covers:

  • Your user profile (email, name, category, credits)
  • All your saved scripts
  • Active session tokens
  • The content of your AI queries in the logs (the email link is anonymized; aggregated counter data is retained without any link to you)

8. Sharing with third parties

We do not sell or rent out data. Personal data is shared only with:

  • Anvil Works (data processor — server and database service, EU)
  • Anthropic (data processor — AI service, USA, Standard Contractual Clauses)
  • Microsoft (identity provider for SSO)
  • Public authorities where required by law

9. Security

The service uses HTTPS for all communication. Session tokens are stored as SHA-256 hashes on the server. Microsoft ID tokens are validated against Microsoft's public keys. The Anvil database is encrypted at rest.

10. Changes to this statement

Changes will be published on this page. Material changes are notified by email to logged-in users.
